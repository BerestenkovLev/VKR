# RehabAI 

Прототип электронной карты реабилитационного обращения, маршрутизации и
ролевого доступа для реабилитационных центров, не подключённых к ЕМИАС.
Часть ВКР «Управление развитием информационного обеспечения сферы
здравоохранения города Москвы».

## ⚠️ Область применимости

Это **демонстратор на синтетических (обезличенных) данных**. Он показывает
логику, структуру данных и архитектурный подход, но **не является
развёрнутой медицинской системой**.

Промышленный контур описывается в работе как **целевая архитектура** и в
прототип не входит:
- хранение и обработка реальных персональных данных по 152-ФЗ / 323-ФЗ;
- защищённое хранилище, СКЗИ, передача по HTTPS/mTLS;
- вход через ЕСИА / корпоративную учётную запись учреждения;
- интеграция с ЕМИАС и городскими информационными системами.

## Архитектура

Клиент-серверное приложение:
- **Backend** — Python, FastAPI, SQLAlchemy 2.0, SQLite. Данные смоделированы
  по ресурсам HL7 FHIR R4 (Patient / Encounter / ServiceRequest), есть
  экспорт обращения в FHIR-бандл.
- **Расчёт приоритета** — прозрачная объяснимая эвристика (см. `app/scoring.py`):
  балл = интенсивность + длительность + возрастной фактор + факторы риска;
  пороги: ≤3 — Низкий, 4–5 — Средний, ≥6 — Высокий. Алгоритм возвращает разбор
  балла, который виден пользователю.
- **Авторизация** — JWT (HS256) + разграничение ролей `pacient` / `specialist`
  (см. `app/auth.py`). Пароли хранятся как PBKDF2-HMAC-SHA256
- **Frontend** — одностраничный интерфейс `app/static/index.html`, раздаётся
  самим сервером (на хостинге это один URL, без установки на клиенте).

```

```


## Демонстрационные учётные записи

| Роль        | Логин     | Пароль       | Доступ                                               |
|-------------|-----------|--------------|------------------------------------------------------|
| Пациент     | `patient` | `patient123` | свои обращения, расчёт приоритета, сохранение записи |
| Специалист  | `doctor`  | `doctor123`  | вся очередь, решения, статистика, сброс данных        |

В интерфейсе вход выполняется кнопками «Войти как пациент / специалист»
(учётки подставляются автоматически). Пароли заданы открыто **только для
демонстрации**; роль определяется учётной записью на сервере.

## Точки API

| Метод | Путь                              | Доступ        | Назначение                                  |
|-------|-----------------------------------|---------------|---------------------------------------------|
| GET   | `/api/health`                     | открыт        | проверка работоспособности                  |
| POST  | `/api/auth/register`              | открыт        | регистрация пациента                        |
| POST  | `/api/auth/login`                 | открыт        | вход, выдача JWT                            |
| GET   | `/api/auth/me`                    | по токену     | текущий пользователь                        |
| POST  | `/api/score`                      | по токену     | расчёт приоритета без сохранения            |
| POST  | `/api/appeals`                    | по токену     | создать обращение (владелец — пользователь) |
| GET   | `/api/appeals`                    | по токену     | специалист — вся очередь, пациент — свои     |
| GET   | `/api/appeals/{id}`               | владелец/спец | одно обращение                              |
| GET   | `/api/appeals/{id}/fhir`          | владелец/спец | экспорт в HL7 FHIR                          |
| POST  | `/api/appeals/{id}/decisions`     | специалист    | решение специалиста, смена статуса          |
| GET   | `/api/stats`                      | специалист    | сводка по очереди                           |
| POST  | `/api/admin/reset`                | специалист    | сброс обращений к демо-данным               |

- интеграция с ЕМИАС и городскими информационными системами;
- вход через ЕСИА, защищённое хранилище секретов, HTTPS/mTLS;
- перенос на защищённое хранилище ПДн в соответствии с 152-ФЗ / 323-ФЗ;
