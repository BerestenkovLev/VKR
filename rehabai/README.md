# RehabAI — серверная часть (этап 1)

Демонстратор электронной карты реабилитационного обращения и маршрутизации
для реабилитационных учреждений, не подключённых к ЕМИАС.

> **Область применения.** Система работает на **демонстрационных (обезличенных)
> данных** и предназначена для показа интерфейса, логики и интеграционной
> совместимости. Промышленный контур — серверное развёртывание в защищённой
> среде, сертифицированное хранилище, СКЗИ, интеграция с ЕМИАС и соответствие
> 152-ФЗ / 323-ФЗ — описывается в работе как **целевая архитектура**. Реальные
> персональные данные в демонстраторе не используются.

## Что это технически

Полноценное клиент-серверное приложение (в отличие от прежнего файла-прототипа на `localStorage`):

- **Бэкенд:** Python + FastAPI, авто-документация Swagger на `/docs`.
- **БД:** SQLite через SQLAlchemy (в целевой архитектуре — PostgreSQL).
- **Данные в форме HL7 FHIR R4** (Patient, Encounter, ServiceRequest) — экспорт через API.
- **Прозрачный расчёт приоритета** (см. `app/scoring.py`): итоговый балл = интенсивность +
  длительность + возрастной фактор + факторы риска; пороги Низкий / Средний / Высокий.
  Расчёт возвращает разбивку по составляющим — алгоритм объясним на защите.

## API

| Метод | Эндпоинт | Назначение |
|-------|----------|------------|
| GET  | `/api/health` | проверка работоспособности |
| POST | `/api/appeals` | создать обращение (приоритет/маршрут считает сервер) |
| GET  | `/api/appeals` | очередь обращений; фильтры `?priority=` и `?status=` |
| GET  | `/api/appeals/{id}` | карточка обращения |
| GET  | `/api/appeals/{id}/fhir` | экспорт обращения в FHIR-бандл |
| POST | `/api/appeals/{id}/decisions` | решение специалиста + обновление статуса |
| GET  | `/api/stats` | сводка по очереди (для дашборда/презентации) |

## Запуск

### Вариант для защиты (без установки на ноутбук)
Бэкенд размещается на хостинге, а на защите открывается **ссылка в браузере** —
на самом ноутбуке ничего ставить не нужно. Конкретные шаги развёртывания на
бесплатный хостинг подготовим на этапе упаковки.
Демо в этом случае зависит от интернета в аудитории, поэтому страхуемся: запись
экрана с рабочим прогоном + мобильная точка доступа как резерв.

### Локальный запуск (для разработки)
Требуется установленный Python 3.10+.
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
# затем открыть http://localhost:8000/docs
```
При первом запуске база создаётся и наполняется демо-данными автоматически.
